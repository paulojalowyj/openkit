# User Stories

| ID | Story |
|---|---|
| US-1 | Como maintainer do OpenKit, quero um padrao unico de nomes em `docs/` para evitar links quebrados e reduzir retrabalho. |
| US-2 | Como usuario do OpenKit, quero encontrar rapidamente a documentacao essencial (com um index curto e objetivo) sem navegar em subpastas. |
| US-3 | Como agente/command do OpenKit, quero contratos de caminhos estaveis para requirements e sprints para que `/plan`, `/tasks`, `/impl` e `/engineer` funcionem de forma previsivel. |
| US-4 | Como contributor em macOS, quero um fluxo de renome seguro (incluindo renomes apenas por capitalizacao) para nao gerar diffs inconsistentes ou conflitos em CI. |
| US-5 | Como CI, quero validacoes que detectem referencias quebradas (paths inexistentes) antes do merge/release. |
